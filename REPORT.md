## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 게임명 | CodeRun (코드런) |
| 장르 | 2D 횡스크롤 무한 러너 |
| 주제 | 취업준비생의 취업 지옥 탈출기 |
| 플랫폼 | 웹 브라우저 (HTML5 Canvas) |
| 해상도 | 가상 캔버스 1280 × 720 px |
| 목표 | 장애물을 회피하며 최대한 오래 달리고, 젤리를 수집해 높은 점수 달성 |

### 게임 흐름

```
로딩 화면 → 메인 메뉴 → 인게임 → 게임오버
                              ↕
                           일시정지
```

플레이어는 자동으로 달리는 캐릭터를 조작해 **점프(단/이중)**와 **슬라이드**로 장애물을 회피하고, 화면에 등장하는 젤리를 수집해 점수를 얻는다. 시간이 지날수록 스크롤 속도가 증가하며 난이도가 상승한다.

## 2. 기술 스택

| 분류 | 기술 |
|------|------|
| 렌더링 | HTML5 Canvas 2D API |
| 언어 | Vanilla JavaScript (ES2022) |
| 모듈 시스템 | ES Modules (`type="module"`) — 번들러 없음 |
| 스타일 | CSS3 (Flexbox, CSS Variables, @keyframes) |
| 폰트 | Courier New (monospace) |
| 에셋 형식 | PNG (스프라이트, 배경, 장애물, 젤리), MP3 (BGM) |
| 저장소 | localStorage (최고기록 영속 저장) |
| 빌드 도구 | 없음 (정적 파일 서빙만 필요) |

### 주요 설계 원칙

- **의존성 제로**: 외부 라이브러리 없이 순수 Web API만 사용
- **모듈 경계 명확화**: 각 파일이 단일 책임을 가지도록 설계
- **싱글톤 패턴**: `AssetLoader`, `AudioManager`, `Game` 인스턴스를 전역 단일 객체로 관리

## 3. 시스템 아키텍처

### 3.1 레이어 구조

```
index.html
 └─ Game (싱글톤)
     ├─ SceneManager         ← 씬 전환, 크로스페이드 관리
     ├─ InputHandler         ← 키보드 / 터치 이벤트 통합
     └─ Scenes
         ├─ MenuScene
         ├─ GameScene        ← 메인 게임 루프
         │   ├─ Player
         │   ├─ ObstacleSpawner → Obstacle[]
         │   ├─ JellySpawner    → Jelly[]
         │   ├─ Background (3-layer parallax)
         │   └─ Ground
         ├─ PauseScene
         └─ GameOverScene
```

### 3.2 씬 생명주기

```
switchScene(key)
  → SceneManager.changeScene()
    → 페이드아웃 (0.25 s)
    → 현재 씬.exit()
    → 새 씬.enter(payload)
    → 0.2 s 블랙홀드
    → 페이드인 (0.25 s)
```

PauseScene은 페이드 없이 `setImmediate()`로 즉시 오버레이된다.

### 3.3 게임 루프

```javascript
requestAnimationFrame → _loop(timestamp)
  dt = min((timestamp - lastTime) / 1000, 0.1)  // 최대 100 ms 캡
  SceneManager.update(dt)
  InputHandler.flush()
  SceneManager.render(ctx)
```

## 4. 기술 명세

### 4.1 렌더링 파이프라인

#### 가상 캔버스 & 레터박스 스케일링

물리 픽셀 해상도에 무관하게 항상 1280 × 720 가상 좌표계를 사용한다. 브라우저 창 크기 변경 시 `setTransform`으로 스케일과 오프셋을 재계산해 레터박스(letterbox) 방식으로 표시한다.

```javascript
scaleX = windowW / 1280
scaleY = windowH / 720
scale  = min(scaleX, scaleY) × DPR      // 비율 유지
offsetX = (windowW × DPR - 1280 × scale) / 2
offsetY = (windowH × DPR - 720  × scale) / 2
```

#### 인게임 렌더 순서 (페인터 알고리즘)

```
1. 배경 그라데이션 (sky: #6a7888 → #3a4858 → #04040e)
2. Far_Background.png   (시차 0.1×)
3. Midground.png        (시차 0.4×)
4. Ground (컨베이어 벨트 + 매트릭스 레인)
5. Foreground.png       (시차 0.7×)
6. 장애물 (Obstacles)
7. 젤리 (Jellies)
8. 플레이어 (Player)
9. 점수 팝업 (ScorePopup)
10. HUD 오버레이
```

#### 에셋 프리로딩

게임 시작 전 `AssetLoader.preload(manifest)` 가 모든 이미지를 `Promise.all`로 병렬 로드한다. 로딩 완료 후 로딩 오버레이가 페이드아웃된다.

```javascript
await AssetLoader.preload({
  ...OBSTACLE_MANIFEST,
  ...PLAYER_MANIFEST,
  ...BACKGROUND_MANIFEST,
  ...JELLY_MANIFEST,
});
```

### 4.2 물리 시스템

#### 좌표계

| 상수 | 값 | 설명 |
|------|----|------|
| `CANVAS.WIDTH` | 1280 px | 가상 캔버스 너비 |
| `CANVAS.HEIGHT` | 720 px | 가상 캔버스 높이 |
| `GROUND_Y` | 600 px | 지면 Y 좌표 (캔버스 하단 120 px) |

#### 플레이어 물리 상수

| 상수 | 값 | 설명 |
|------|----|------|
| `GRAVITY` | 1800 px/s² | 중력 가속도 |
| `JUMP_FORCE` | −900 px/s | 1단 점프 초기 속도 |
| `DOUBLE_JUMP_FORCE` | −720 px/s | 2단 점프 초기 속도 |
| `FAST_FALL_VY` | 650 px/s | 공중 슬라이드 하강 초기 속도 |
| `AIR_SLIDE_GRAV` | 0.25× | 공중 슬라이드 중 중력 배율 |
| `SLIDE_DURATION` | 0.5 s | 슬라이드 최소 지속 시간 |
| `MAX_JUMPS` | 2 | 최대 연속 점프 횟수 (이중 점프) |

#### 점프 궤적 분석

단일 점프(v₀ = 900 px/s, g = 1800 px/s²)의 최대 체공 시간:

```
t_top = 900 / 1800 = 0.5 s  (정점 도달)
t_land = 2 × 0.5 = 1.0 s    (착지)
h_max = 900² / (2 × 1800) = 225 px  (지면 기준 최대 상승 높이)
```

이 값을 기준으로 젤리 아크 패턴과 슬라이드 장애물 높이를 설계했다.

#### 플레이어 상태 머신

```
RUNNING ──[점프]──→ JUMPING ──[2단 점프]──→ DOUBLE_JUMPING
    ↑                   │                         │
    └──[착지]────────────┘─────────────────────────┘
    │
    ├──[슬라이드키]──→ SLIDING ──[타이머 만료]──→ RUNNING
    │
    └──[피격]──→ HIT ──[무적 종료]──→ RUNNING
```

**공중 슬라이드 (Air Slide)**
점프 중 슬라이드 키 입력 시:
- 히트박스를 즉시 SLIDE_H(70 px)로 전환
- `vy = max(현재vy, 650)` 으로 하강 가속
- 중력 배율 0.25× 적용 → 일정 속도로 빠르게 하강
- 착지 시 지상 슬라이드로 자연스럽게 연속

#### 히트박스 명세

| 상태 | 너비 | 높이 |
|------|------|------|
| 기립 / 점프 | 80 px | 130 px |
| 슬라이드 | 80 px | 70 px |

### 4.3 입력 처리

`InputHandler`는 키보드와 터치를 통합 관리한다.

| 입력 | 동작 |
|------|------|
| Space / ↑ / W | 점프 (Just Pressed) |
| ↓ / S | 슬라이드 (Just Pressed: 발동, Held: 슬라이드 연장) |
| P / ESC | 일시정지 |
| ` / B | 디버그 모드 토글 |
| 터치 1핑거 | 점프 |
| 터치 2핑거 홀드 | 슬라이드 |

**점프 우선 처리**: 슬라이드 키 홀드 상태에서 점프 키를 누르면 같은 프레임에 점프만 처리되고 슬라이드 재발동을 억제한다.

```javascript
if (input.jumpJustPressed) player.jump();
if (input.slideJustPressed && !input.jumpJustPressed) player.slide();
```

### 4.4 충돌 감지

AABB(Axis-Aligned Bounding Box) 방식을 사용한다. `margin` 값으로 판정 영역을 안쪽으로 수축시켜 시각적 겹침 허용량을 조절한다.

```
판정 조건:
(a.x + margin) < (b.x + b.width  - margin)  AND
(a.x + a.width  - margin) > (b.x + margin)   AND
(a.y + margin) < (b.y + b.height - margin)  AND
(a.y + a.height - margin) > (b.y + margin)
```

| 충돌 쌍 | margin |
|---------|--------|
| 플레이어 ↔ 장애물 | 8 px |
| 플레이어 ↔ 젤리 | 2 px |

**멀티 히트박스**: `DeadlineBanner`는 아치형 구조물로, 내부 통과 영역(아치 아래)을 제외하기 위해 복수의 히트박스를 정의한다.

```
DeadlineBanner 히트박스:
  ┌─────────────────────────┐ ← 상단 배너 면 (0, 0, 1082, 330)
  │                         │
  │    ╔═══════════════╗    │
  │    ║  중앙 기둥      ║    │ ← (100, 330, 850, 110)
  └────╚═══════════════╝────┘
```

### 4.5 장애물 스포너

#### 난이도 3단계 페이징

| 단계 | 경과 시간 | 스폰 간격 | 출현 패턴 |
|------|-----------|-----------|-----------|
| Early | 0 ~ 20 s | 1.8 ~ 2.8 s | 단일 장애물 |
| Mid | 20 ~ 60 s | 1.2 ~ 1.9 s | J+J 쌍, S→J 콤보 |
| Late | 60 s~ | 1.0 ~ 1.8 s | J+J+J 3연속, S→J+J |

#### 최소 안전 거리 (MSD)

연속 장애물 간 플레이어가 반드시 회피 가능한 최소 간격이다.

| 전 → 후 | MSD | 근거 |
|---------|-----|------|
| 슬라이드 → 점프 | 700 px | 슬라이드 0.5 s 완료 후 점프 가능 |
| 점프 → 슬라이드 | 900 px | 착지 후 슬라이드 자세 전환 여유 |
| 점프 → 점프 | 80 px | 물리적 공간 확보만 필요 |

#### 크로스패턴 전환 타이머

패턴 간 전환도 타이머로 제어해 불가능한 구간을 방지한다.

| 전환 | 최소 대기 |
|------|-----------|
| 점프 장애물 → 슬라이드 장애물 | 1.5 s |
| 슬라이드 장애물 → 점프 장애물 | 0.9 s |

### 4.6 젤리 스포너

#### 젤리 타입 명세

| 타입 | 컨셉 | 점수 | 등장률 | 크기 |
|------|------|------|--------|------|
| java | Java 아이콘 | 1 pt | 55 % | 48 × 48 |
| python | Python 곰젤리 | 3 pt | 25 % | 32 × 48 |
| c | C 포인터 | 4 pt | 10 % | 48 × 48 |
| mysql | MySQL 돌고래 | 5 pt | 8 % | 48 × 48 |
| git | 커밋 잔디 | 10 pt | 2 % | 48 × 48 |

#### 스폰 패턴

| 패턴 | 개수 | 배치 방식 |
|------|------|-----------|
| Line | 12개 | 지면 위 수평 일렬, 간격 58 px (JELLY_SIZE + 10) |
| Arc | 최대 12개 | 점프 포물선 궤적을 따라 58 px 간격 배치 |
| Cluster | 8개 | 반지름 65 px 원형 배열 |

**Arc 패턴의 장애물 회피 배치**: 스폰 시 Arc X 범위 내 장애물을 탐색하여, 겹치는 젤리의 Y 좌표를 `장애물.y - JELLY_SIZE - 10`으로 올려 장애물 위에 자연스럽게 드리운다.

```
스폰 간격: 0.7 ~ 1.5 s
```

### 4.7 점수 시스템

#### 점수 구성

```
총점 = 거리 점수 + 젤리 점수
거리 점수 = (이동 거리 / 100m) × 1 pt
젤리 점수 = 젤리 기본점 × 콤보 배율
```

#### 콤보 시스템

| 연속 수집 | 배율 |
|-----------|------|
| 1개 | 1.0× |
| 2 ~ 3개 | 1.5× |
| 4 ~ 5개 | 2.0× |
| 6개 이상 | 3.0× |

콤보는 마지막 젤리 수집 후 **3.0 초** 내에 다음 젤리를 수집해야 유지된다.

#### 최고기록

`localStorage`에 `'coderun_best'` 키로 영속 저장된다. 신기록 달성 시 게임오버 화면에서 파티클 이펙트와 골드 배지가 표시된다.

### 4.8 오디오

Web Audio API를 사용하지 않고 HTML `<Audio>` 객체를 이용한 간단한 BGM 관리를 구현했다.

```
BGM: Midnight_Keyboard_Rush.mp3 (무한 반복)
기본 볼륨: 0.5
```

브라우저 자동재생 정책(Autoplay Policy) 대응: `play()` Promise가 reject될 경우 첫 번째 사용자 입력(`keydown` / `pointerdown`) 이벤트에서 재시도한다.

| 이벤트 | 동작 |
|--------|------|
| MenuScene 진입 | BGM 시작 (최초 1회) |
| PauseScene 진입 | BGM 일시정지 |
| PauseScene 종료 | BGM 재개 |

## 5. 게임 설계

### 5.1 난이도 설계

#### 월드 스피드 증가 곡선

```
초기 속도:  480 px/s  (8 px/frame @ 60fps)
최대 속도: 1200 px/s  (20 px/frame @ 60fps)
증가 단계:  +30 px/s  매 5초마다
총 증가 구간: (1200 - 480) / 30 = 24단계 = 120초 후 최고속
```

점프 체공 시간은 중력·초기속도 고정값이라 스피드와 무관하게 약 1.0초를 유지한다. 따라서 속도가 빨라질수록 같은 시간 안에 더 넓은 구간을 이동해야 하므로 반응 속도 요구가 증가한다.

#### 슬라이드 장애물 회피 가능성 검증

```
서있는 플레이어:
  상단 y = GROUND_Y - NORMAL_H = 600 - 130 = 470
  → 여유 margin 6 px → 판정 상단 = 476

슬라이드 중 플레이어:
  상단 y = GROUND_Y - SLIDE_H  = 600 - 70  = 530
  → 여유 margin 6 px → 판정 상단 = 536

NaggingBubble 하단 y = GROUND_Y - 570 + 470 = 500
  → 서있으면 476 < 500 → 충돌 ✓
  → 슬라이드하면 536 > 500 → 통과 ✓

이중 점프 최고 도달 y ≈ 600 - 130 - 225 - 144 = 101
NaggingBubble 상단 y = GROUND_Y - 570 = 30
  → 101 > 30 → 뛰어넘기 불가 ✓
```

### 5.3 젤리 설계

취업 준비생이 알아야 할 기술 스택을 컨셉으로 삼았다.

- **java (55%)**: 가장 흔하고 안정적인 백엔드 기본기 — 낮은 점수의 기본 재화
- **python (25%)**: 다재다능한 스크립트 언어 — 중간 점수
- **c (10%)**: 컴공의 매운맛, 필수 역량 — 획득 어려운 고점수
- **mysql (8%)**: 구조화된 데이터 적재 — 희귀 고점수
- **git (2%)**: 취준생의 성실함 — 최고 점수·최저 등장률 (역전의 무지개 잔디)

---

### 5.4 캐릭터 설계

**스프라이트 애니메이션**: PNG 스프라이트 시트에서 소스 rect를 추출해 `ctx.drawImage` 9인수 형식으로 그린다. 히트박스(Entity)와 드로우 크기는 분리 관리되며, 바텀-센터 기준으로 정렬된다.

| 애니메이션 | 프레임 수 | FPS | 루프 |
|-----------|---------|-----|------|
| run | 8 | 6 | ✓ |
| jump | 7 (회문 배열) | 6 | ✓ |
| slide | 2 | 4 | ✓ |
| gameover | 4 | 6 | ✗ |

점프 애니메이션은 `[1→2→3→4→3→2→1]` 회문 배열로 상승·하강 동작을 자연스럽게 표현한다.

## 6. UI/UX 설계

### 6.1 비주얼 컨셉

전체적으로 **쿠키런 for Kakao** 스타일의 따뜻한 다크 UI를 채택했다.

| 화면 | 배경색 | 포인트 색 |
|------|--------|-----------|
| 메인 메뉴 | `#0e0c0a` (웜 다크) | 골드 `#ffd060`, 앰버 `#ffaa20` |
| 인게임 | 배경 PNG + 그라데이션 | 네온 그린 `#00ff88`, 레드 `#e94560` |
| 게임오버 | `#0d0b09` + 레드 비네트 | 골드 `#ffd700`, 오렌지 `#ff9020` |

### 6.2 HUD 구성

```
┌─────────────────────────────────────────────────────┐
│ [HP ♦♦♦]                          000070    ← 점수 │
│                                    37 m      ← 거리 │
│                                    COMBO ×3.0       │
│                                    [콤보 타이머 바]  │
│                                                     │
│  (게임 화면)                                         │
│                                                     │
│ SPC/↑ jump  ↓ slide  P pause  F1 debug    ← 힌트  │
└─────────────────────────────────────────────────────┘
```

- **HP**: 자기소개서 아이콘 (최대 3개)
- **점수**: 6자리 제로패딩 (`000070`)
- **콤보 바**: 남은 콤보 시간을 색상이 변하는 프로그레스 바로 표시

### 6.3 씬별 인터랙션

| 씬 | 조작 | 설명 |
|----|------|------|
| 메인 메뉴 | ↑↓ ENTER | 메뉴 선택 |
| 인게임 | Space/↑ ↓ P/ESC | 점프, 슬라이드, 일시정지 |
| 일시정지 | ↑↓ ENTER ESC/P | 메뉴 선택, 즉시 재개 |
| 게임오버 | ↑↓ ENTER ESC | RETRY / MAIN MENU |

### 6.4 로딩 화면

CSS `@keyframes`로 로딩 바가 0 → 100% 애니메이션된다. 이미지 프리로드 완료 후 `opacity: 0` 전환으로 페이드아웃된다.
